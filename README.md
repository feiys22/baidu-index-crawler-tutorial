## 快速开始（5 分钟跑通）

### 第一步：安装依赖

```bash
pip install qdata
```

### 第二步：获取百度 Cookie

这是最关键的一步！Cookie 相当于你的"登录凭证"，百度靠它识别你的身份。

1. 用 Chrome 打开 https://index.baidu.com，确认已登录（右上角有头像）
2. 按 **F12** 打开开发者工具
3. 点击 **Console（控制台）** 标签
4. 输入以下代码，按回车：

```javascript
document.cookie
```

5. 复制输出的整段字符串

**看不懂？详细步骤见 [Cookie 获取完整指南](docs/02-cookie-guide.md)**

### 第三步：填入代码

打开 `src/crawl_baidu_index.py`，找到这一段：

```python
COOKIE_LIST = [
    '''请在这里填入第一个百度账号的完整Cookie''',
    '''请在这里填入第二个百度账号的完整Cookie''',
]
```

把刚才复制的 Cookie 粘贴进去，**注意用三引号包裹**（三引号详解见 [这里](docs/03-cookie-format-explained.md)）。

### 第四步：运行

```bash
cd src
python crawl_baidu_index.py
```

### 第五步：等待数据

程序会依次爬取每个城市的数据，每个城市保存为一个 CSV 文件在 `src/` 目录下。

**144 个城市大约需要 14-16 小时（建议晚上跑，第二天看结果）。**

## 常见问题

**Q：Cookie 是什么？为什么需要它？**

A：Cookie 是你登录百度后服务器给你的"身份凭证"。我们的爬虫程序借用了这个凭证，不需要密码就能获取数据。详细原理见 [Cookie 指南](docs/02-cookie-guide.md)。

**Q：Cookie 多久过期？**

A：通常 7-30 天。如果程序开始报 `status: 10018` 错误，说明 Cookie 过期了，重新获取填入即可。

**Q：会被封号吗？**

A：正常使用情况下不会。百度主要限流而不是封号。如果请求太频繁会触发临时限制，降低频率或换 Cookie 即可。

**Q：能爬多少数据？**

A：取决于你的 Cookie 数量和网络状况。3 个 Cookie 轮询，144 个城市 × 52 个关键词，大约 14-16 小时可完成。

**Q：数据是实时的吗？**

A：百度指数通常有 1-2 天的延迟。比如今天是 5 月 2 日，数据可能只到 4 月 30 日。

**Q：能爬历史数据吗？**

A：可以！修改代码里的 `START_DATE` 和 `END_DATE` 即可。例如 `'2025-01-01'` 到 `'2025-12-31'` 就能爬 2025 年全年的数据。
